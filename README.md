# Kindraw

Kindraw is a free multi-drawing whiteboard built for people who want a hosted workspace without the overhead of a traditional SaaS suite.

It keeps the editor fast and sketch-first, then adds the product layer around it:

- GitHub login
- automatic draft creation when an authenticated user lands on `/`
- folders and nested folders for organizing drawings
- remote autosave
- public read-only links generated by the Kindraw API
- GitHub-aware realtime presence, so collaborators are shown with their logged-in identity

## What Kindraw is

Kindraw is not trying to be a generic whiteboard SaaS clone. The current product direction is drawing-first:

- you open the app and immediately land in a new drawing
- workspace management stays close to the canvas
- sharing is explicit: either a public read-only link or a live realtime session
- the free tier is built around Cloudflare Pages, Workers, D1 and R2

## Stack

- Frontend: React + Vite
- Editor engine: Kindraw canvas shell built from the upstream fork
- Realtime canvas: built-in room collaboration flow
- API: Cloudflare Worker
- Database: Cloudflare D1
- Blob storage: Cloudflare R2
- Auth: GitHub OAuth
- Package manager: Yarn 1

## Repository layout

- `excalidraw-app`: main web application
- `packages/excalidraw`: editor package inherited from the upstream fork
- `workers/api`: Kindraw backend on Cloudflare Workers
- `examples`: upstream integration examples kept from the fork

## Local development

Install dependencies:

```bash
yarn install
```

Create local Worker secrets:

```bash
cp workers/api/.dev.vars.example workers/api/.dev.vars
```

Fill `workers/api/.dev.vars` with your GitHub OAuth app credentials:

```env
GITHUB_CLIENT_ID="..."
GITHUB_CLIENT_SECRET="..."
```

Apply the local D1 migration:

```bash
yarn --cwd workers/api wrangler d1 migrations apply kindraw-db --local
```

Run the Worker:

```bash
yarn dev:worker
```

Run the frontend:

```bash
yarn start
```

Open:

- `http://localhost:3001/`
- `http://localhost:8787/` for the local API

### GitHub OAuth for local development

Configure your GitHub OAuth app with:

- Homepage URL: `http://localhost:3001`
- Authorization callback URL: `http://localhost:8787/api/auth/callback/github`

## Deployment

Kindraw is deployed Cloudflare-first:

- frontend to Cloudflare Pages
- API to Cloudflare Workers
- schema migrations to Cloudflare D1
- drawing blobs to Cloudflare R2

GitHub Actions included in this repo:

- `.github/workflows/kindraw-ci.yml`
- `.github/workflows/deploy-kindraw-frontend.yml`
- `.github/workflows/deploy-kindraw-d1.yml`
- `.github/workflows/deploy-kindraw-api.yml`

Required GitHub secrets:

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

## Product notes

- Public links are read-only and generated by the Kindraw API.
- Realtime collaboration still uses the upstream room model.
- When users are authenticated with GitHub, realtime presence displays their GitHub-backed identity.
- The root route is optimized for flow: authenticated users go straight into a fresh drawing.
